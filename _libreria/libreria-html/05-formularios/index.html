Formularios en HTML

Los formularios permiten a los usuarios enviar informació

Trabajar con un formulario es un proceso de dos partes. La primera parte corresponde a recibir los datos por parte del usuario.

La segunda parte está a cargo de un "script" que se encarga de procesar la información y responder a ella. (Esta étapa está fuera de los alcances de este curso.)

Elemento <form>

La etiqueta <form> corresponde al primer paso para crear un formulario en HTML

Su función es servir de contenedor de todos los elementos de ingreso de información

Una etiqueta tradicional se construye así

<form action="scripts/script.php" method="POST" id="miFormulario"> </form>

Descripción

La etiqueta <form> acepta una serie de atributos, en este ejemplo utilizamos tres básicos.

action
La etiqueta "action" corresponde a una URL. Esta URL es la encargada de procesar la información que se envía a través del formulario
method
Este atributo especifica la forma como la información se envía al servidor.
id
El atributo id es común para todas las etiquetas y corresponde a un valor único para identificar el elemento.

Elementos <fieldset> y <legend>

La etiqueta <fieldset> es un elemento que permite agrupar lógicamente dentro de un formulario.

La etiqueta <legend> provee de un subtítulo a un <fieldset>.

Ejemplo

<form action="scripts/script.php" method="POST" id="miFormulario"> <fieldset> <legend>Datos Personales</legend> </fieldset> </form>

Elemento <label>

La etiqueta <label> se utiliza para asociar un texto descriptivo con su campo de ingreso de información asociado.

Esta etiqueta es especialmente útil para mejorar la accesibilidad de nuestros formularios.

Ejemplo

<form action="scripts/script.php" method="POST" id="miFormulario"> <legend>Datos Personales</legend> <label for="tu-nombre">Nombre:</label> <input type="text" id="tu-nombre" /> </form>

Campos de texto simples

Los campos de texto son uno de los elementos más comunes en la red. Nos sirven para ingresar una palabra o una linea de texto.

Para ingresar un campo de texto simple utilizamos la etiqueta <input>

La etiqueta <input> se utiliza en conjunto con su atributo "type". El cual define la etiqueta como un campo de texto.

Ejemplo

<form action="scripts/script.php" method="POST" id="miFormulario"> <legend>Datos Personales</legend> <label for="tu-nombre">Nombre:</label> <input type="text" value="nombre" size="40" maxlength="10" /> </form>

Descripción

Los atributos que la etiqueta contiene son:

type
El valor de este atributo define el tipo de datos que el campo de texto recibe, en este caso su valor es "text".
value
El atributo "value" especifica el texto que por defecto almacena el campo de texto. Este texto es enviado al servidor en caso que ningún otro se ingrese
size
Los navegadores crean por defecto campos de texto de 20 caracteres de longitud, a través de este atributo podemos cambiarlo. (Recuerda que también podemos hacerlo desde CSS.)
maxlength
Los campos de texto pueden contener infinitos caracteres, en ocaciones es de utilidad limitar esta cantidad, para eso podemos utilizar el atributo maxlength

Los atributos y valores nuevos son:

readonly
El valor "readonly" indica que el valor de este campo no se puede modificar mientras este valor exista.
type="password"
El valor "password" del atributo type funciona de la misma manera que el valor "text", excepto que reemplaza las letras por asteríscos o circulos.

Áreas de texto

Cuando es necesario que los usuarios ingresen más de una linea de texto, podemos utilizar la etiqueta <textarea>.

A diferencia que la etiqueta <inpuy>, la etiqueta <textarea> requiere una etiqueta de apertura y una de cierre, todo lo que se ingrese entre estas etiquetas corresponderá al texto por defecto de la caja de texto.

Ejemplo

Area de Texto
<fieldset> <legend>Area de Texto</legend><label>Ingresa tu comentario</label> <textarea name="comentario" cols="70" rows="8"></textarea> </fieldset>

Descripción

Los atributos y valores nuevos son:

row
Especifica el número de filas que el area de texto mostrará. Si el usuario supera este límite, el campo creará automáticamente un barra de navegación al costado.
cols
Indica el ancho del campo de texto medido en el número de caracteres que puede almacenar.

Subida de archivos

Los navegadores pueden recibir documentos y transmitirlos desde el disco duro del usuario.

Para crear un campo que permita elegir un archivo desde el computador utilizamos la etiqueta <input> y el atributo "type" pero con el valor "file".

Ejemplo

<label>Suba su archivo:</label> <input type="file" />

Checkbox

Los checkbox se crean a través de la etiqueta <input> y el atributo "type" pero con el valor "checkbox".

Puedes crear uno o un grupo de campos de selección asignándoles el mismo valor a su atributo "name".

La característica más importante de estos elementos, es que el usuario puede seleccionar más de uno a la vez.

Ejemplo

Checkbox - Me gusta:
<fieldset> <legend>Checkbox - Me gusta:</legend> <input type="checkbox" name="opcion" checked="checked" /><label> Comer</label> <input type="checkbox" name="opcion" /><label> Dormir</label> <input type="checkbox" name="opcion" /><label> Ver televisión</label> <input type="checkbox" name="opcion" /><label> Leer</label> </fieldset>

Descripción

Los atributos y valores nuevos son:

type="checkbox"
El valor indica que el elemento <input> es del tipo "checkbox".
checked="checked"
Podemos seleccionar un elemento por defecto a través de este atributo.

Botones de radio

Los botones de radio se crean a través de la etiqueta <input> y el atributo "type" pero con el valor "radio".

A diferencia que los botones de selección, el atributo "name" es obligatorio. Esto porque los botones de radio siempre estarán agrupados lógicamente y el usuario sólo podrá elegir uno.

Ejemplo

Radio Buttons
<fieldset> <legend>Radio Buttons</legend> <input type="radio" name="opcion" value="uch" /> <label>Universidad Católica</label> <input type="radio" name="opcion" value="cc" /> <label>Colo Colo</label> <input type="radio" name="opcion" value="uc" /> <label>U. de Chile</label> </fieldset>

Importante

Podrás notar que cada boton de radio tiene el mismo valor para el atributo "name", esto es así porque este valor permite al servidor identificar la selección del usuario.

El valor del atributo "value" sin embargo, ya que esta es la información que almacena la selección del usuario.

Menú de selecciones

Los menú de selecciones son una opción para entregar a los usuarios una lista de opciones, desde la cual pueden elegir una preferencia.

Ejemplo básico:

Selecciónes
<fieldset> <legend>Selecciónes</legend> <label>Selección normal</label> <select name="colores"> <option value="r">Rojo</option> <option value="g">Verde</option> <option value="b">Azul</option> </select> </fieldset>

Descripción

Las etiquetas son:

select
La etiqueta <select> indica que el elemento despliegará una serie de opciones para el usuario
name="colores"
El valor del atributo name se envía al servidor para reconocer la selección hecha por el usuario.
option y "value"
Cada elemento <option> contiene un texto que representa la opción, el valor del atributo "value" es el valor que se envía al servidor.

Más ejemplos de menu de selecciones

Menú Expandido

Para visualizar más de una opción en el menu, podemos utilizar el atributo "multiple" con el valor "multiple".

Para especificar cuántas opciones son visibles utilizamos el atributo "size" y a su valor le asignamos la cantidad de elementos visibles.

<fieldset> <label>Selección multiple</label> <select name="normal" multiple="multiple" size="4"> <option>Rojo</option> <option>Azul</option> <option>Verde</option> <option>Amarillo</option> <option>Negro</option> </select> </fieldset>
Agrupar opciones

Podemos crear grupos conceptuales de opciones a través de la etiqueta <optgroup>

Esta etiqueta requiere del atributo "label" para definir el título del grupo

<fieldset> <select name="grupos" multiple="multiple" size="6"> <optgroup label="Vegetales"> <option>Lechuga</option> <option>Tomate</option> </optgroup> <optgroup label="Animales"> <option>Vacuno</option> <option>Pollo</option> </optgroup> </select> </fieldset>
